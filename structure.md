# Laravel Project Structure Explanation

## Root-Level Directories

### /app - Application Core

Contains all your business logic - models, controllers, services, and other custom classes.

**What typically goes in it:**
- Models (database tables as PHP classes)
- Controllers (handle HTTP requests)
- Services (reusable business logic)
- Middleware (request filtering)

### /resources - Frontend Assets

Everything related to the user interface:
- `views/` - Blade templates (HTML with PHP)
- `css/` - Stylesheets
- `js/` - JavaScript files

### /routes - URL Definitions

Defines which URLs map to which controller methods:
- `web.php` - Routes for browser-based pages (HTML responses)
- `api.php` - Routes for API endpoints (JSON responses)

### /database - Database Files

Everything related to the database schema:
- `migrations/` - Database table definitions
- `seeders/` - Sample data
- `factories/` - Test data generators

### /tests - Automated Tests

Code that automatically verifies your application works correctly:
- `Unit/` - Tests for individual classes (models, services)
- `Feature/` - Tests for entire features (login flow, signup flow)

### /public - Web Server Root

The only folder accessible directly via browser:
- `public/index.php` - Entry point
- `public/css/` - Compiled CSS
- `public/js/` - Compiled JavaScript

### /storage - Application Storage

Files generated by the application:
- `logs/` - Error logs
- `framework/sessions/` - Session files
- `framework/views/` - Compiled Blade templates

### /vendor - Third-Party Code

Laravel framework and all installed packages (via Composer). Do NOT commit to Git.

### /config - Configuration Files

Application settings for different services (database, mail, cache, etc.)
